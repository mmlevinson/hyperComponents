A commonly used interface component is an `Accordion`.   

The user sees a list of titles with an icon opposite the title text.  Clicking on the title smoothly reveals additional content directly below the title.  The icon rotates to indicate the open/closed state of the control.

An example accordion control written in Hyperscript is shown below.

## Description

Accordions conserve screen real estate while still allowing considerable content to be displayed when needed.

To implement this control two parts are needed:

1.  A header (with a clickable title)
2.  A body (with hidden content that can  be revealed)

In the collapsed state, the height of the  body is  set to 0.  In response to a click event in the header, the body smoothly transitions to an open state by setting a final height and a transition animation. From a programming standpoint, the key is calculating the body's final height based on the proposed content.  

If too much content is loaded into the body, a huge vertical shift occurs on the page which is distracting.   Thus it is best to set a maxxHeight after which extra content simply scrolls vertically.

## Features

1.  Clicking the header toggles the body open or closed.
2.  An indicator icon  rotates 180 degrees to indicate open/closed state
3.  Revealing body content is animated smoothly using a Hyperscript `transition`
4.  A maxHeight prop prevents the body height from exceeding a set value.
5.  Tailwind classes for the header, body, and active header (when opened) can be passed in via props to style the entire component from outside.
6.  Content for the body is passed into the default slot and thus can be styled from where the accordion is used. 

## Components

This control is comprised of two Astro components:

1.  `AccordionGroup_A` which functions as a wrapper for all the individual items, and
2.  `AccordionItem_A` which consists of a button for the header and a div for the body content.

All of the source code, including the scripts and styling, is found in the [Code](#code) section below.

## Props

Props are used to configure both Accordion components from where they are used.  This strategy allows full customization without having to rewrite the  source code itself.

All  props except strings unless otherwise specified.

### AccordionGroup_A


```js

interface Props {
	closeSiblings?:boolean,
	styles?:string
}

const {
	closeSiblings = false, 
	styles
} = Astro.props

```



#### closeSiblings

If opening an accordion should cause  all other accordions to close then pass `true` to the `closeSiblings` prop.  This will cause a custom `closeOpenAccordions` event to be fired.  All accordions except the one sending this event will be then closed.

The default value is `false`.

#### styles

This prop takes a string containing any Tailwind classes of your choosing.   These classes will be merged with default styles (which you find in the [Code](#code) section below.

TW classes passed in the `styles` prop only affect the whole group and thus should be mostly positioning values such as width, margins, etc.

### AccordionItem_A


```js

interface Props {
	id:string,
	caption:string,
	prefix?:string
	maxHeight:number
	headerStyles?:string
	bodyStyles?:string
	activeHeaderStyles?:string
}
const {
	id, 
	caption, 
	prefix='- ',
	maxHeight=300,
	headerStyles,
	bodyStyles,
	activeHeaderStyles = 'bg-orange-100 dark:bg-blue-950',
} = Astro.props
```

#### id

Each item should have a unique id to prevent naming collisions with other items.

#### caption

The caption is the text that appears in the accordion header.

#### prefix

This props accepts is a character or short string that is prepended to the caption.  It acts as a tick mark. The default is `- `

#### maxHeight

This prop sets the maximum height of the accordion body.  The default is `300px`.

You can pass any value but must include the CSS units for the script to apply correctly.  Valid examples would be like :

1.  '300px'
2.  '40rem'
3.  '25%'

:::danger
Passing a value without appending a valid CSS unit will fail.
::::

#### headerStyles

This prop takes a string containing any Tailwind classes of your choosing.   These classes will be merged with the default  styles for the `header` section of AccordionItem_A (which you find in the [Code](#code) section below.  

Use this prop to override or add styling to the `header` section of the accordion.

#### bodyStyles

This prop takes a string containing any Tailwind classes of your choosing.   These classes will be merged with the default  styles for the `body` section of AccordionItem_A (which you find in the [Code](#code) section below.

Use this prop to override or add styling to the `body` section of the accordion.

#### activeHeaderStyles

This prop takes a string containing any Tailwind classes of your choosing.   These classes will be added to the `header` when the accordion is open.

Use this prop to add styling to the `header` only when it is open.


## Styling

Both the AccordionGroup_A and AccordionItem_A components can be styled with suitable TW utility classes from outside of the component code base using props.   This allows the appearance of both the group and individual items to be customized without having to modify the source code.

You can override the default styling by sending additional Tailwind classes via [props](#code) which are then merged with the default styles and any conflicts resolved with a 'last wins' rule.

To improve readability, the Tailwind classes and Hyperscripts are both extracted from the markup and encapsulated into  Astro `tw` and `hs` local variables within the [Component Script](https://docs.astro.build/en/basics/astro-components/#the-component-script) which are then referenced in the html markup. 

The advantage of this strategy is to allow long run-on strings to be formatted as concatenated, multi-line strings which aids readability and code maintenance.

## Transitions

For this accordion, the `transition` method of Hyperscript is utilized to specify the CSS properties, duration and timing function used to smoothly reveal the accordion body. 

What makes this Hyperscript example unique is the interpolation of the component props into the script.  

To make the values of external props available to the script, the script text is extracted from the markup and placed into a local `hs` constant within the Astro Component Script (i.e. frontmatter).   The script itself is referenced into the html markup as would be for any value in the frontmatter/

```js "script={hs}"

<li class={tw.wrapper}>

	<!-- Accordion Header -->
	<button 
		id={id} 
		class={tw.accordionHeader} 
		script={hs}>
			<span>{prefix}{caption}</span>
			<!-- TwirlIcon -->
			<svg id={`${id}-twirl-icon`} class={tw.icon} fill='currentColor' viewBox='0 0 20 20' xmlns='http://www.w3.org/2000/svg'>
					<path fill-rule='evenodd' d='M5.293 7.293a1 1 0 011.414 0L10 10.586l3.293-3.293a1 1 0 111.414 1.414l-4 4a1 1 0 01-1.414 0l-4-4a1 1 0 010-1.414z' clip-rule='evenodd'></path>
				</svg>
	</button>

	<!-- Accordion Body -->
	<div 	id={`${id}-body`} class={tw.accordionBody}>
		<slot />
	</div>

</li> 

```

Now, using back-tick strings to wrap the entire Hyperscript  allows for string interpolation of the props directly into the script itself.  Dynamic values can thus be passed into the script itself via props creating another level of functionality.

Here is a snippet of code from the accordion HyperScript where the `maxHeight` and `openDuration` component props are relayed  into the script using string interpolation which are then used to configure the `transition` for the accordion body.

```astro  "${maxHeight}" "${openDuration}"
	`transition the #{:accordionBody}'s *height to ${maxHeight}  using 'all ${openDuration} ease-in' `
```

The syntax for the `transition` method of Hyperscript must be exact or it will not work.  After `using` clause you provide a quoted string analogous to how a transition in vanilla CSS would be written.  If you leave out the quotes, the transition will fail silently.

##  Usage

Accordion controls are often used in FAQ or Info sections of a web site.  The advantages of using an Accordion is to hide the majority of the content until the user needs to see it.  By just presenting the content title in the header the user is prompted to explore the content by clicking on the header.   Meanwhile, the extra text in the body of the Accordion is hidden until requested.

This, of course, makes Accordions an ideal choice for lazy loading content. To enhance the UX, content for the Accordion body does not need to be loaded until selected by the user.   Using HTMX, lazy loading is easily done by listening for the click event and then reaching out to the server for html which is loaded into the Accordion body.   The transition animation is done in response to the htmx:load event.
