A common use case for a custom dropdown is to present a Pallet of colors.   Clicking a color then updates the styling of some user element.  

## Examples

Two examples are presented which  illustrate a drop down menu with a grid of color swatches.  For illustration purposes we are using Tailwind's color families.

The first example uses the [ClickDropdownMenu_A](/components/menus/click_a/) parent component to wrap a custom [PalletGrid](#palletgrid) component.   The code block used to present  this example is listed to follow.

### Description

This hyperComponent utilizes either the  [ClickDropdownMenu_A](/components/menus/click_a) or [HoverDropdownMenu_B](/components/menus/hover_b) parent components presented in earlier.

The default slot for the dropdown component(s) receives a [PalletGrid](#palletgrid) custom component.   The PalletGrid takes an `orientation` prop which returns either a horizontally or vertically oriented grid of color swatches.  

Each swatch is comprised of  a [PalletItem](#palettitem) component  which accepts a `twColor` and `hexCode` prop.   The `twColor` is the Tailwind color class and the `hexCode` is the equivalent hexadecimal value of the color.   

### Features

-The trigger button receives `caption` prop so the dropdown's  appearance is only indicated by its icon

-The Tailwind logo icon is passed into the `leftIcon` slot, with appropriate styling classes applied.

-The `duration` prop is set to '180ms' but can be any value.

-Hovering over an individual swatch reveals a tooltip displaying the Tailwind color name and corresponding hex value.

-Clicking a swatch dismisses the dropdown smoothly and then displays an alert showing the color name and hexidecimal value of the selection.

-A class of `.colorPallet` is added to the `buttonStyles` prop of the parent component so that the Hyperscript associated with each swatch  can send it a custom event which triggers closure of the dropdown after a selection is made.

The dropdown is dismissed by:

###### 1. Tapping the trigger button again
###### 2. Clicking outside the dropdown
###### 3. Mouse leaving the dropdown
###### 4. Tapping any color swatch
###### 5. Pressing the Escape key when the trigger button is focused

In the hover activated example, HS behaviors are installed into the  `<button>` and dropdown container from a `<script>` tag in the [HoverDropdownMenu_B](/components/menus/hover_b) component. This behavior adds tracking of the mouseenter and mouseleave system events in order to toggle dropdown visibility based on hover state. 


## Components

### PalletGrid

This component is passed into the default slot of each dropdown parent component.  The PalletGrid takes an `orientation` prop. The default orientation is `horizontal`. In the hHorizontal orientation a total of 22 Tailwind color families are placed into columns  across a grid layout.   Since Tailwind itself does not offer a 'grid-cols-22' utility class and  'grid-cols-[22]' did not work, a  `<style>` tag was used in the PalletGrid component to create the 22 column layout.

The Vertical orientation is organized differently with the same color shaed in column with a different color family in each row.

Here is the code for the PalletGrid component.  The horizontal and vertical pallets are imported and a JSX style ternary expression used to decide which orientation is returned depending on the value of the  `orientation` prop.

```js title="PalletGrid.astro"
---
import PalletGridH from "./PalletGridH.astro"
import PalletGridV from "./PalletGridV.astro"

interface Props {
	orientation:string
}

const {orientation = "horizontal"} = Astro.props

---

<div >

	{orientation === 'horizontal' ? <PalletGridH/> : <PalletGridV/>}

</div>


```

### PalletItem

The essential component rendering a single swatch is the `PalletItem` comprised of a simple `<span>` with a Hyperscript attribute that fires a custom event closing the dropdown after a selection.

The PalletItem component accepts two props.   The `twColor prop is the name of the Tailwind color such as 'red-500'.    The `hexCode` prop is the corresponding hexidecimal value for that color.  

A function in the component script capitalizes the first letter of the color name which is then used in the `title` prop to display for a tooltip that is displayed when the user hovers over the color swatch.

Since the swatch is styled using background colors, the 'bg-' prefix is stripped before the name is displayed.

Finally, a Hyperscript attribute on each `PalletItem` listens for a click event and then sends the `closeDropdown` custom event back to the parent button.  To make this happen, a `.colorPalett` class is explicitly added to the `<button>` of each dropdown component by passing this class name into the `buttonStyles` prop of the dropdown component. 

After the dropdown is transitioned off the screen, an alert pops up showing the name and hex value the user selected.

To set the background color of the swatch a simple back-tick interpolated expression is used to combine the default styles with the `twColor` prop.   Since there are no default background colors, twMerge() not needed since there will not be any conflicts with the initial styles. 

:::tip
Notice the space char between the variables composing the class list in the example code.  Since you are essentially concatenating strings to build the class list, leaving out this space character would concat two adjacent classes which breaks the styling."
:::

```js title="PalletItem.astro" "span" "twColor" "hexCode"
---
interface Props {
	twColor: string
	hexCode:string
}

const {twColor, hexCode } = Astro.props

function capitalizeFirstCharacter(str) {
    if (!str) return str;
    return str.charAt(0).toUpperCase() + str.slice(1);
}

const color = capitalizeFirstCharacter(twColor.replace(/^bg-/, ''))

const tw = {
	palletItem: 'size-3 rounded-sm overflow-hidden ring-1 ring-slate-600 hover:scale-125',
}

---

<span 
	class={`${twColor} ` + tw.palletItem} title={`${color} (#${hexCode})`}
	script="on click send closeDropdown to the .colorPallet then settle then alert( 'You selected ' + my @title)">
</span>




```


## Code

Use the Tabs below to explore all the code used to create these examples.  

To apply  one of these palettes in your Astro project, copy/paste the code from either the Click or Hover activated examples into the Astro file where it will be used.

You will also need to copy/paste the PalletGrid, PalletGridH, PalletGridV, and  PalletItem code and put them into separate `.astro` files.

The import file paths in the sample code are not specific for your project structure so they will need to be adjusted depending on where in your src folder you place the `.astro` files.

You should change the action associated with clicking a color swatch by updating the Hyperscript in the `<span>` of the PaletteItem.   You  probably don't want to just show an alert when the swatch is clicked  but actually perform some more useful action.   Adjusting script in the PaletteItem propogates the your intended action to every color swatch.  

However, before your custom action is triggered, you should  `send closeDropodown to the .colorPallet then settle` to smoothly dismiss the dropdown before the user is taken away to something else.
