A common use case for a custom dropdown is to present a Pallet of colors.   Selecting a color then updates the styling of some user element.  

## Examples

### Description

This hyperComponent utilizes the both  `ClickDropdownMenu_A` or `HoverDropdownMenu_B` parent components presented in earlier.

The choices passed to the component default slot, in this case, are another custom Astro component which encapsulates a single color choice into a `<span>` formatted as a small rectangular color swatch by setting its background color to one of the Tailwind color classes.   In addition, the Tailwind designated name and hexadecimal value are attached to the  `title` attribute of the `<span>` so they will appear as a tooltip when the user hovers over the color swatch.

### Features

1. The trigger button has no caption text so is indicated to the user only by its icon
2. The Tailwind logo is passed into the `leftIcon` slot, with appropriate styling classes to size it.
3. The `duration` attribute is set to '180ms' to make the menu open and close a bit faster.
4. Hovering over an individual swatch reveals a tooltip with the Tailwind color name and corresponding hex value.
5. Clicking a swatch dismisses the dropdown smoothly and then displays an alert showing the color value selected.
4. A class of `.colorPallet` is added to the `buttonStyles` prop of the parent component so that the Hyperscript in each PalletItem can send it a custom event which closes the dropdown after a selection is made.

The dropdown is dismissed by:

###### 1. Tapping the trigger button again
###### 2. Clicking outside the dropdown
###### 3. Mouse leaving the dropdown
###### 4. Tapping any color swatch
###### 5. Pressing the Escape key when the button is focused

In the hover activated example, HS behaviors are installed into the  `<button>` and dropdown container from a `<script>` tag in the HoverDropdownMenu_B component. This behavior adds tracking of the mouseenter and mouseleave system events to toggle dropdown visibility based on hover state.

### PalletGrid

This component is the overall wrapper for all the swatchess and is placed into the default slot of each dropdown menu component.

 To include all the Tailwind default theme colors, a 22 column grid is needed.    Since Tailwind itself does not offer a 'grid-cols-22' and the 'grid-cols-[22]' did not work, a component `<style>` tag was used to create the 22 column grid and make is responsive with media queries.

Inside the PalletGrid are 22 separate wrapper `<div>`s, each containing the individual swatches.  The wrappers are styled as `flex flex-col` to present the variants of each TW base color in a column layout.

### PalletItem

The essential component rendering a single swatch is the PalletIte comprised of a simple `<span>` tag as detail below.

The designated name of the specific Tailwind color variant is passed into the PalletItem component through the `twColor` prop, for example, "red-500".   The corresponding hex code value is passed through the `hexCode` prop.  

A function in the component script capitalizes the first letter of the color name which is then used in the `title` prop to display (along with the hex value) as a tooltip when the user hovers over the PalletItem.  

Since the swatch is styled using background colors, the 'bg-' prefix is stripped before the name is displayed.

Finally, a Hyperscript attribute on each PalletItem listens for a click event and then sends the `closeDropdown` custom event back to the parent button (which has the .colorPallet class)

After the dropdown is transitioned off the screen, a alert pops up showing the Name and hex value the user selected.

```js title="PalletItem.astro" "span"
---
interface Props {
	twColor: string
	hexCode:string
}

const {twColor, hexCode } = Astro.props

function capitalizeFirstCharacter(str) {
    if (!str) return str;
    return str.charAt(0).toUpperCase() + str.slice(1);
}

const color = capitalizeFirstCharacter(twColor.replace(/^bg-/, ''))

const tw = {
	PalletItem: 'size-3 rounded-sm overflow-hidden ring-1 ring-slate-600 hover:scale-125',
}

---

<span 
	class={`${twColor} ` + tw.PalletItem} title={`${color} (#${hexCode})`}
	script="on click send closeDropdown to the .colorPallet then settle then alert( 'You selected ' + my @title)">
</span>



```

 Tailwind offers 12 variants for each theme color.  Each variant is represented by one PalletItem.  In turn, each PalletItem is a wrapper for a `<span>`.   Each span contains HS code which closes the dropdown after a swatch is chosen.
