Astro

Astro enters the arena of application build tools with an emphasis on speed and small bundle sizes

Initially offered as a static site generator, Astro now provides a full suite of server-rendering capabilities as well.

Static sites can be generated from Markdown files and shipped with no embedded Javascript code making them load quickly in the client's browser

Developers can create Web components simply by creating a .astro file. Any JavaScript written in the Component Script portion of the .astro file is compiled and executed during initial render of the component

This allows for individuality of behavior for components by bundling the code and markup together in the same file

Server-rendering of pages can be automatic or individualized by setting a simple constant in the source file

Astro also designates islands of behavior allowing the developer to designate which DOM elements are hydrated on the client side and for which events this takes place

Web components built with other popular frameworks can be used within any Astro project

Routing in an Astro project if file based so not specific configuration is needed by the developer to create specific routes for specific content

Any file within the 'pages' folder becomes a route, including markdown files or custom API routes (ie endpoints)

Custom API endpoints are easy to create avoiding the need to write a separate backend server in node.js/Express or Django. To create a custom endpoint, the only requirement is to export a GET or POST constant in any astro file within the pages folder. A route for that endpoint is then automatically created during the build process and available to any HTTP request sent to that route.
