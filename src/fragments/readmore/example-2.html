TailwindCSS is broad set of utility classes which can be used to style any DOM element.

The concept is to provide granular control of element styling using a consistent set of class names which are then converted to actual CSS by the Tailwind compiler.

The advantage of using utility classes includes

  • Rapid prototyping of styles and appearance
  • Mobile first responsive design
  • Avoding media queries by using simple prefixes (sm, md, lg, xl, etc) to represent different screen breakpoints
  • Locality of Behavior for setting element styling

Using Tailwind can be as simple as including a script tag pointing to the Tailwind CDN.

For production, Tailwind should be installed in your node_modules folder via npm or other package manager and then configured by updating the properties of the tailwind.config.mjs file in your source folder.

Implementing Tailwind styling is as simple as adding class names to any html element. Since the styling changes appear immediately in the resulting DOM, prototyping is quicker and more flexible than relying on independent CSS files that are separated from the markup they control.
